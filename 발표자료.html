01 — BACKGROUND

왜 만들었나 — 기획 배경

고객센터의 전화접수 현황은 "얼마나 들어와서, 얼마나 현장으로 넘어갔는지(필터링율)"가 핵심 지표입니다. 문제는 이 숫자를 매주 사람이 엑셀을 열어 손으로 집계했다는 점입니다.

🧮

반복되는 수작업 집계

1차필터·2차필터 탭을 매주 열어 행을 세고, 현장인계 건을 빼고, 필터링율을 계산하는 과정을 손으로 반복했습니다.

🗂️

흩어진 데이터

전화상담 전체(1차필터)와 AS·현장(2차필터)이 서로 다른 탭·기준으로 존재해, 한눈에 보는 통계가 없었습니다.

🔒

개인정보 노출 위험

원본에는 요청자명·카드번호·차량번호 등 민감정보가 섞여 있어, 그대로 공유·집계하기 어려웠습니다.

💡 핵심 아이디어 — 시트는 입력 창구로만 쓰고, 집계·중복제거·통계는 모두 Supabase(DB)가 SQL로 계산하게 한다. 사람은 시트에 입력만 하고, 숫자는 대시보드가 실시간으로 보여준다.
02 — PROBLEMS

해결해야 했던 문제점

단순히 시트를 DB로 옮기는 것을 넘어, 데이터를 신뢰할 수 있게 만드는 과정에 난관이 있었습니다.

① 고유 ID 없는 1차필터 중복

전화상담(1차필터)에는 접수번호 같은 고유 ID가 없어, 행번호 키로 적재하면 정렬·중간삽입 때 같은 건이 중복(684 → 868건)으로 늘어났습니다.

② 제각각인 날짜 포맷

접수일시가 yyyymmddHHmmss 같은 8·12·14자리 패킹 숫자로 들어와, 주차·월 집계를 하려면 먼저 표준 날짜(ISO)로 정규화해야 했습니다.

③ 개인정보 마스킹

차량번호·차량ID는 ***로 마스킹하고 요청자명·카드번호·자유텍스트는 아예 저장하지 않도록, 적재 단계에서 걸러야 했습니다.

④ 집계 기준의 일관성

"현장인계 건은 필터 집계에서 제외" 같은 규칙을 매번 손으로 적용하면 사람마다 숫자가 달라져, 규칙을 SQL에 고정할 필요가 있었습니다.

03 — SOLUTION

어떻게 개선했나 — 4단계 자동 파이프라인

시트 입력 → GAS 적재 → Supabase 집계 → Vercel 대시보드. 사람의 손은 맨 앞 한 번뿐입니다.

① 구글시트에 입력

상담사가 1차필터(전화상담 전체)·2차필터(AS/현장) 탭에 평소처럼 데이터를 입력합니다.

② GAS가 자동 적재 (10분 주기 · 전체 새로고침)

installSyncTrigger가 10분마다 실행 → DB를 비우고 시트를 다시 적재. 적재 중 차량번호·차량ID 마스킹, 날짜 ISO 정규화, 배치 내 중복 제거를 수행합니다.

③ Supabase가 SQL로 집계

뷰·함수(cs_week_label)가 일/주/월 단위로 자동 계산. "현장인계 제외" 같은 규칙이 SQL에 고정돼 누가 보든 같은 숫자가 나옵니다.

④ Vercel 대시보드가 직접 조회

Next.js 웹앱이 anon 키로 집계 뷰만 읽어, KPI·분해 차트·월간 추이를 실시간으로 보여줍니다.

BEFORE
  • 매주 엑셀을 열어 손으로 집계
  • 현장인계 제외·필터율을 수기 계산
  • 사람마다 다른 숫자, 검증 어려움
  • 민감정보가 그대로 섞인 원본 공유
→
AFTER
  • 입력만 하면 10분 내 자동 반영
  • 집계 규칙을 SQL에 고정 — 항상 같은 숫자
  • 주차·월 통계를 웹에서 실시간 확인
  • 마스킹·미저장으로 개인정보 보호
🔁 전체 새로고침 방식 — 1차필터는 고유 ID가 없어 증분 동기화 시 중복이 생기므로, 매 실행마다 비우고 다시 적재해 시트와 DB를 정확히 일치시킵니다.

04 — IMPACT

기대 효과

반복 수작업을 없애고, 누가 보든 같은 신뢰할 수 있는 숫자를 실시간으로 제공합니다.

⏱️

집계 시간 제로화

매주 반복하던 수기 집계가 사라지고, 시트 입력 후 10분이면 대시보드에 자동 반영됩니다.

✅

일관된 통계
